KEY PRINCIPLES IMPLEMENTED
─────────────────────────────────────────────────────────────────────────────

1. NO DIRECT SUPABASE IN PAGES
   ✓ All database access isolated to repositories
   ✓ Pages only call entity query factories
   ✓ Features only call entity mutations

2. REPOSITORY PATTERN
   ✓ Base repository class with CRUD + helpers
   ✓ Entity-specific repositories extend base
   ✓ Complex queries handled in repositories
   ✓ Consistent error handling

3. DOMAIN MODELS
   ✓ Independent domain entities
   ✓ Business logic encapsulated
   ✓ Value objects and types
   ✓ Entity class methods (isActive, canEdit, etc.)

4. DTO MAPPING
   ✓ Database DTOs isolated
   ✓ Mapper converts DTO ↔ Domain Entity
   ✓ Type transformations (string → Date, etc.)
   ✓ No database schema leaked to UI

5. REACT QUERY FACTORIES
   ✓ Centralized query key definitions
   ✓ Reusable query hooks
   ✓ Reusable mutation hooks
   ✓ Eliminated query/mutation duplication

6. CENTRALIZED AUTHORIZATION
   ✓ Permission enum (type-safe)
   ✓ Role ↔ Permission mapping
   ✓ Automatic permission checks in mutations
   ✓ Organization-scoped access

7. AUTOMATED AUDIT LOGGING
   ✓ Enum-based audit actions
   ✓ Automatic logging in mutations
   ✓ User context + timestamp tracking
   ✓ Non-blocking error handling

AUDIT LOGGING SYSTEM

Audit Actions (Enum):

  - DOCUMENT_UPDATED
  - DOCUMENT_DELETED
  - DOCUMENT_APPROVED
  - TASK_ASSIGNED
  - TASK_DELEGATED
  - (and 20+ others)

Automatic in Mutations:
  ✓ Logs action with user context
  ✓ Captures timestamp
  ✓ Records changes made
  ✓ Tracks old/new values
  ✓ Non-blocking (won't break app)

Manual Usage:
  await auditService.success(context, AuditAction.DOCUMENT_CREATED, id);
  await auditService.failure(context, AuditAction.DOCUMENT_CREATED, id, error);
